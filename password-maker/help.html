PasswordMaker Options

Master Password

A single, unencrypted password which, when concatenated with the URL textbox, becomes the message from which an unique password is generated.

Save Master Password

If checked, the password you entered in the Master Password textbox will be stored (in an encrypted fashion) on the local hard drive. The next time you use PasswordMaker, the Master Password textbox will automatically be pre-populated with the last value it contained.

Use l33t

Specifies when to apply l33t (also known as leetspeak), if at all. l33t can be applied not at all, to the unencrypted master password and URL only (i.e., before the hashing function has been applied), to the generated password only (i.e., after the hashing function has been applied), or to both the unencrypted master password and URL as well as the generated password (i.e., before and after applying the hashing function.

l33t Level

The complexity of l33t (also known as leetspeak) applied after the encrypted password has been generated. A side-effect of using l33t (in this version of PasswordMaker) is that all upper-case characters are converted to lower-case. Hash Algorithm

From this list, the encyrption method used to generate passwords can be selected. All algorithms are one-way hash functions, also known as message digests and fingerprints.

URL Components

These checkboxes enable you to easily select which parts of the current URL should be included in the URL textbox. You don't have to use these checkboxes at all; you can simply type the URL you like directly into the URL textbox. We recommend only using the domain portion of a URL because the rest of the URL is subject to change by whomever owns the site you are using. Even the protocol may change from http:// to https://, or vice-versa. The only time you should consider using another part of the URL is when two or more passwords are needed for the same domain; i.e., the site does not implement single sign-on authentication for its various applications.

Protocol

When checked, the protocol portion of the current URL is included in the URL textbox. In the URL http://foo.bar.mozilla.org:8080/this/is/the/path.html#anchoraway?lang=en&id=123, PasswordMaker considers http:// the protocol.

Subdomain(s)

When checked, all subdomains of the current URL -- if any -- are included in the URL textbox. In the URL http://foo.bar.mozilla.org:8080/this/is/the/path.html#anchoraway?lang=en&id=123, PasswordMaker considers foo.bar the subdomains.

Domain

When checked, the domain of the current URL is included in the URL textbox. In the URL http://foo.bar.mozilla.org:8080/this/is/the/path.html#anchoraway?lang=en&id=123, PasswordMaker considers mozilla.org the domain.

Port, path, anchor, query parameters

When checked, the port, path, anchor, and query parameters -- if any -- of the current URL are included in the URL textbox. In the URL http://foo.bar.mozilla.org:8080/this/is/the/path.html#anchoraway?lang=en&id=123, PasswordMaker considers :8080/this/is/the/path.html#anchoraway?lang=en&id=123 the port, path, anchor, anf query parameters.

URL

The URL which, when concatenated with the Master Password textbox, becomes the message from which an unique password is generated. We recommend only using the domain portion of a URL because the rest of the URL is subject to change by whomever owns the site you are using. Even the protocol may change from http:// to https://, or vice-versa. The only time you should consider using another part of the URL is when two or more passwords are needed for the same domain; i.e., the site does not implement single sign-on authentication for its various applications.

Length of Generated Password

Here you specify the length, in characters, for the generated password.

Generated Password

This textbox displays the generated password.

Copy to Clipboard

Copies the contents of the Generated Password textbox to the clipboard (not available in the on-line version).
